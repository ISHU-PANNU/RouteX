# RouteX — Smart Courier & Logistics Platform
## Technical Architecture & Implementation Blueprint

RouteX is a logistics and courier management platform built to resolve last-mile delivery inefficiencies. By incorporating optimized route generation, simple turn-by-turn route navigation via Google Maps deep-links, and dual-verification mechanisms (QR code scans + OTP handshakes), RouteX delivers enterprise-grade reliability and transparency to customers, couriers, and administrators.

This repository contains the complete **Software Architecture Blueprint and Product Specification** generated by the Principal Software Architect. Development is ready to start immediately.


## 🛠️ Technology Stack
- **Frontend Core:** React.js Single Page Application (SPA)
- **Styling UI:** Tailwind CSS (utility-first framework)
- **Backend API:** FastAPI (ASGI Python, asynchronous endpoints)
- **Database Engine:** MySQL 8.0+ (relational transactions)
- **ORM mapping:** SQLAlchemy (Repository patterns + migration control via Alembic)
- **Validations:** Pydantic (data parsing & constraints validation schemas)
- **Security:** JSON Web Tokens (JWT) + Role-Based Access Control (RBAC)


Features

### 1. Delivery Route Planner
Compares the original delivery order with an optimized route and displays estimated distance and travel-time savings. Google Maps API is used strictly for coordinates geocoding and fetching distance/duration matrices, avoiding non-deterministic AI overhead and expensive black-box services.

### 2. Route Navigation
Enables delivery agents to view their daily roster of assigned deliveries, click "Navigate with Google Maps" to launch optimized turn-by-turn routing directly in Google Maps, and update shipment statuses during deliveries.

### 3. QR-Based Parcel Intake
Ensures parcels are scanned correctly at the warehouse and loaded onto the courier's vehicle. Scanning the parcel’s physical barcode matches its unique secure QR hash to the database, transitioning the status to `Out For Delivery`.

### 4. OTP Handoff Verification
A secure 6-digit numeric OTP is shared with the customer upon dispatch. The delivery agent must input this matching code in their mobile-responsive portal. Only a matching code transitions the package status to `Delivered`.
